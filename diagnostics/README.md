# Diagnostics

这个目录用于归档问题定位、故障分析和案例复盘。这里按产品或代码库继续分层，避免把一次具体问题定位混进长期机制专题里。

## 范围

- 线上或本地复现的问题现象。
- 日志、目录结构、配置和源码证据。
- 根因分析、规避方式和后续检查项。
- 与长期机制笔记之间的关联。

## 已归档案例

- [GaussDB/openGauss 表空间回放软链接冲突分析](gaussdb/tablespace-symlink-replay.md)

## 归档规则

1. 先记录现象和结论，再补定位过程。
2. 机制解释可以链接到 `../postgres/` 或 `../gaussdb/`，不要在案例里重复堆太多通用概念。
3. 同类问题变多后，再在对应子目录新增索引文件。
